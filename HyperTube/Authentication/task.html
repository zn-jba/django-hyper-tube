Description

It all started as a hosting for a bunch of friends, but now it is getting popular. It is time to help the users interact with the server closer with the help of user authentication. In the future, this functionality will also help you grant the registered users more rights.

Objectives

The service needs two pages in order to authenticate users: signup and login. On the signup page, users register their accounts, and on the login page, they pass the authentication by their username and password.

You can use django.contrib.auth.forms.UserCreationForm for your signup handler. For the login handler, use django.contrib.auth.forms.AuthenticationForm or django.contrib.auth.views.LoginView.

If the login request is successful, redirect the user to the previous page.

If you start the application on your computer with the command python manage.py runserver, the signup page should be available at the address localhost:8000/signup/. The login page should be at localhost:8000/login/.
